# Iteration Whiteboard

**Change request:** with the lighter new theme some components have light text making it unreadable fix this issues, also the public page for the drop is too complex make it more simple and easy to use, make inline sub navs with tabs if needed, design for mobile first.. the logo looks like a drop of blood. maybe we should come up with a better name and logo that fits this idea

**Subtasks planned:** 4

1. **BrandStrategist**: The current 'Drop' name + red droplet logo reads like a blood drop, which clashes with a warm, surprise-generosity product. Propose a refined brand direction: evaluate whether to keep 'Drop' (with a non-droplet mark, e.g. a confetti burst, wrapped gift, paper-plane flock, group-chat bubble cluster, surprise party popper) OR rename to something warmer (candidates like 'Rally', 'Huddle', 'Pile On', 'Surprise', 'Chipin', 'GroupHug', 'Showering'). Pick ONE final direction with rationale, define the new logo concept in words (shape, motion, colors — must work with the existing light/friendly theme), update the tagline if the name changes, and write brand.md or update the existing brand doc with: final name, logo concept, primary/secondary colors (ensure AA contrast on light backgrounds), voice examples. Deliver a concrete spec the Designer can execute — do NOT leave multiple options open.
2. **Designer**: Two deliverables. (1) Build the new logo SVG per BrandStrategist's spec (inline React component + favicon). It must NOT look like a blood droplet. Replace every reference to the old droplet mark across the app (header, footer, favicon, email templates, OG images, loading states). (2) Audit the app for readability issues introduced by the lighter theme — find every place with light-on-light text (muted foregrounds on white, white text on pastel gradients, low-contrast placeholder/helper text, disabled states, badge text, toast text, email preview text) and specify the corrected Tailwind classes / CSS variables (target WCAG AA, 4.5:1 for body). (3) Redesign the PUBLIC drop page mobile-first: current page is too complex. Simplify to a single hero (recipient name, story, progress), then inline tabbed sub-nav (e.g. 'Story' | 'Contributors' | 'Activity' | 'Share') so the page isn't one endless scroll. Tabs must be thumb-reachable, sticky on scroll, and collapse gracefully on desktop. Deliver a written redesign spec + component-by-component class changes the Engineer can implement directly (file paths + before/after).
3. **Engineer**: Implement everything from BrandStrategist and Designer. Specifically: (a) swap in the new logo SVG + update favicon + update all brand name references (app header, footer, <title>, meta tags, email templates, OG image generator, auth screens, empty states, package.json name if appropriate). (b) Apply the contrast fixes across every component the Designer flagged — update Tailwind classes / CSS variables so no light-on-light text remains; verify in both the marketing/landing page and the authenticated dashboard. (c) Rebuild the public drop page (app/pages/drop/[id] or equivalent) mobile-first with an inline tabbed sub-nav as specified — Story / Contributors / Activity / Share. Keep the animated chart + copy-share-link features from the prior iteration, just reorganize them into tabs. Use shadcn Tabs or a lightweight custom tabs primitive; make sure tabs are keyboard-accessible and sticky under the hero on mobile. Run `npm run build` until it exits 0. No TODOs, no console.log noise.
4. **QA**: Verify on a mobile viewport (375px) and desktop: (1) no light-on-light unreadable text anywhere — walk the landing page, auth, organizer dashboard, create-drop flow, and public drop page; flag any contrast failures with file:line. (2) New logo appears everywhere the old droplet was, and nothing still references the old mark. (3) Public drop page tabs work: tap targets ≥44px, sticky on scroll, keyboard nav, correct tab state preserved on refresh (if implemented via URL). (4) `npm run build` passes. Report pass/fail per item; do not mark complete if any regressions remain.

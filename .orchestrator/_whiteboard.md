# Iteration Whiteboard

**Change request:** verify that text campaign is supported you should be able to copy a link and it renders a nice branded for phone messages that they can click and be part of it follow?

**Subtasks planned:** 2

1. **Engineer**: Verify and complete SMS/text-message link-preview support for drop invite links. Specifically: (1) Audit the drop page route (the signed contributor link) and confirm it renders server-reachable Open Graph and Twitter Card meta tags: og:title, og:description, og:image (1200x630), og:url, og:type=website, twitter:card=summary_large_image, plus theme-color. Since this is a Vite SPA, static index.html meta won't carry per-drop content — either (a) add a lightweight Lambda@Edge / CloudFront function / dedicated Lambda route that serves pre-rendered HTML with per-drop OG tags when the user-agent is a link-preview bot (facebookexternalhit, Twitterbot, Slackbot, iMessage, WhatsApp, etc.), or (b) pre-bake generic branded meta into index.html and a dynamic per-drop OG image URL. (2) Extend the existing @vercel/og Lambda (used for email images) with a /og-image?dropId=... endpoint that returns a 1200x630 PNG showing: Drop wordmark, 'You're invited to a Drop for {firstName}', progress bar, goal, contributor count — match the email image style. (3) On the organizer dashboard, add a prominent 'Copy link for text' button next to the existing share controls with a toast confirming 'Link copied — paste into iMessage, WhatsApp, or any group chat'. Include subtle helper text: 'Previews as a branded card in most messaging apps.' (4) Make sure the signed link itself is SMS-friendly: short enough to not wrap awkwardly, no tracking junk, https. If the domain is long, note whether a short-link layer is needed (do not build one unless trivial). Do not add TODOs — fully implement. Run npm run build until clean.
2. **QA**: Validate the text-message sharing experience end-to-end. (1) Create a test drop, copy the link via the new 'Copy link for text' button, and paste into: iMessage (iOS), Android Messages (RCS), WhatsApp, Slack, Discord, and Twitter/X DM — verify a branded card with the Drop-for-{name} OG image, title, and description appears in each (document any that fail to unfurl). (2) Use the Facebook Sharing Debugger and Twitter Card Validator (or equivalent curl with User-Agent: facebookexternalhit/1.1 and Twitterbot) to confirm meta tags are served and og:image returns a valid 1200x630 PNG under ~1MB. (3) Confirm the link is copyable on desktop AND mobile (navigator.clipboard fallback works on iOS Safari). (4) Confirm the preview does NOT leak the recipient's full name, Venmo handle, or any surprise-spoiling detail beyond what's already intended for invitees — preserve the conspiratorial-surprise feel. (5) Report findings with screenshots; flag any gaps for Engineer to fix.
